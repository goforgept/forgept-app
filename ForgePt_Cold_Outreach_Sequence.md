## Extra proof points / value-prop bank

Swap these into any email above, or use to personalize per recipient. All pulled directly from what the Designer tool actually does:

- **Manufacturer-agnostic device library** — the core hook. Devices span cameras, access control, intrusion, AV/paging, IT networking, HVAC/electrical/DAS, and door operators, from any manufacturer, all placeable on the same floor plan. Most integrators have only used single-brand tools (a manufacturer's free design software, for example) and haven't seen a cross-trade, cross-brand version.
- **Rack Builder** — any rack type (4-post, 2-post, enclosed, wall-mount, shelf), U-position layout, device labels and accessories, room-by-room. Pairs naturally with the agnostic device library as the "plot it and rack it" pitch.
- **Design-to-proposal-to-job continuity** — device placement feeds the BOM and proposal pricing directly; once a job is sold, it can be tracked through completion in the same platform. This is the "one tool, not four" story — good for prospects who are clearly juggling separate software today.
- **Client Overview PDF** — clean, branded, one-page-per-floor-plan package for client presentations.
- **Shop Drawings package** — title sheet, symbol legend, device schedule, floor plans, cable schedule, all auto-generated and consistently formatted.
- **As-Built package** — IP addresses, MAC addresses, switch ports, serial numbers, patch panel labels tracked per device — useful for integrators who do commissioning/handoff docs.
- **CSV BOM export** — devices, components, cable footage, vertical rises, ready for pricing/ordering.
- **Live client review links** — client can leave pinned notes on the drawing and approve electronically, optionally PIN-protected, with an expiration date.
- **DORI Analysis Report** — automatic IEC 62676-4 detection/observation/recognition/identification distance calculations per camera. Strong secondary point once a camera-heavy prospect is already engaged — save it for a reply or a live demo rather than the cold open.

## Usage notes

- **Personalize the first line.** Whatever specific detail you have about the recipient's business matters more than anything else in the sequence — it's the difference between "cold" and "clearly did their homework." Don't skip it if you have the info.
- **Don't fabricate stats.** I intentionally didn't put in specific numbers like "saves 40% of your time" since there's no real data behind that yet. If you get a few customers using this, swap in a real quote or a real before/after number in Email 2 or 3 — specific, true numbers convert far better than generic claims, but a made-up stat is a liability if anyone asks about it.
- **Cadence:** the day spacing above is a reasonable default (0 / 3-4 / 10-12 / 18-21), but adjust based on whatever outreach tool you're using and how the list responds.
- **CAN-SPAM basics:** since this is commercial email to a list, include a physical mailing address and a working unsubscribe/opt-out in your actual sends (most outreach tools — Instantly, Apollo, Smartlead, etc. — handle this automatically, but double check).
- **Track replies, not just opens.** Open rates are unreliable with modern email clients; watch reply rate and positive-reply rate per subject line variant to see what's actually working, then kill the losing subject line after ~50-100 sends per variant.
